# הפקת הוצאות קבועות אוטומטית דרך n8n

השרת של מטבח החסד חושף endpoint מאובטח שמפיק את ההוצאות הקבועות (שכירות, חשמל, שכר…)
לפי **היום בחודש** שנקבע לכל הוצאה. n8n קורא ל-endpoint הזה פעם ביום, וה-endpoint עצמו
מחליט אילו הוצאות מגיעות להיום ומפיק רק אותן.

> למה n8n ולא cron פנימי? שרת ה-Render החינמי נרדם אחרי ~15 דק'. קריאה חיצונית מ-n8n
> גם מעירה אותו וגם אמינה בתזמון.

---

## שלב 1 - להגדיר את המפתח הסודי בשרת (Render)

בדשבורד Render, בשירות **`birkat-shmuel`** → Environment, הוסיפי משתנה:

```
CRON_SECRET = <מחרוזת אקראית ארוכה - למשל 32 תווים>
```

לייצור מחרוזת אקראית אפשר: `openssl rand -hex 24` (או כל מחולל סיסמאות).
בלי המשתנה הזה, ה-endpoint מחזיר 503 ומסרב לפעול - זו הגנת ברירת מחדל.

> ⚠️ אחרי הוספת env ב-Render השרת עולה מחדש אוטומטית.

## שלב 2 - לייבא את ה-Workflow ל-n8n

1. ב-n8n: **Workflows → Import from File** ובחרי את `n8n-recurring-expenses-cron.json`.
2. ה-workflow כולל 3 צמתים:
   - **כל יום ב-06:00** - Schedule Trigger יומי (אפשר לשנות שעה).
   - **הפק הוצאות של היום** - HTTP POST ל-`/api/cron/run-due`.
   - **נוצרו הוצאות?** - בודק אם נוצר משהו (ל-branch של התראה, אופציונלי).

## שלב 3 - להגדיר את המפתח ב-n8n

ה-workflow קורא את המפתח ממשתנה סביבה של n8n בשם `BIRKAT_CRON_SECRET`
(בכותרת `x-cron-secret`). שתי אפשרויות:

- **מומלץ:** להגדיר ב-n8n משתנה סביבה `BIRKAT_CRON_SECRET=<אותו ערך כמו ב-Render>`
  (ב-self-hosted: ב-`.env` של n8n או ב-docker-compose). ואז לא נוגעים ב-workflow.
- **חלופה:** לפתוח את צומת ה-HTTP ולהחליף את הערך
  `={{ $env.BIRKAT_CRON_SECRET }}` במחרוזת המפתח עצמה (פחות מומלץ - המפתח נשמר גלוי ב-workflow).

## שלב 4 - הפעלה ובדיקה

1. **Execute Workflow** ידני פעם אחת כדי לבדוק. תגובה תקינה נראית כך:
   ```json
   { "month": "2026-07", "created_count": 2, "skipped_count": 1, "created": [...], "day": 14 }
   ```
2. אם קיבלת **401** - המפתח לא תואם. **503** - לא הוגדר `CRON_SECRET` ב-Render.
3. הפעילי את ה-workflow (Active) כדי שירוץ יומית.

## פרטי ה-Endpoints (לעיון)

| Endpoint | מה עושה | מתי להשתמש |
|---|---|---|
| `POST /api/cron/run-due` | מפיק תבניות שה-`day_of_month` שלהן = היום, לחודש הנוכחי | **התזמון היומי** (זה מה שה-workflow עושה) |
| `POST /api/cron/run-month` | מפיק את **כל** התבניות הפעילות לחודש (ללא סינון יום) | גיבוי חד-פעמי אם החמצת ימים |

- אימות: כותרת `x-cron-secret: <CRON_SECRET>` (או `?secret=` ב-query).
- **Idempotent:** קריאה חוזרת באותו יום/חודש לא משכפלת הוצאות - בטוח להריץ שוב.
- ימים 29–31 בחודש ממופים ל-28 (ה-`day_of_month` מוגבל ל-1–28).
- כפייה לבדיקה: `{ "day": 5, "month": "2026-07" }` ב-body מפיק כאילו היום ה-5 ביולי.
